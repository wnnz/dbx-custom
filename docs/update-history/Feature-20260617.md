## 功能概览

### 1. SQL 字段提示与缓存刷新优化

- 修复 `WHERE` 等条件位置无法优先提示当前 SQL 引用表字段的问题。
- 当本地补全结果已存在但引用表字段尚未加载时，会等待字段元数据加载后再展示补全结果。
- 执行 DDL 成功后，会清理对应连接或数据库的 SQL 补全缓存，避免字段变更后继续提示旧字段。
- 表设计器保存已有表结构后，只清理当前表的字段和外键补全缓存，避免影响其它表的补全缓存。
- 新建表时会清理当前数据库级补全缓存，确保表名提示能获取新表。
- 新增测试覆盖 `WHERE` 字段提示和表级补全缓存失效行为。
